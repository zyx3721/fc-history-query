# fc-history-query v1.0.2 版本更新日志

**发布日期**：2026-07-21
**版本类型**：功能更新

---

## 版本概述

v1.0.2 新增查询操作终端日志，便于运维人员追踪查询任务的来源、执行结果与耗时，同时避免在查询过程中因页面自动轮询产生大量重复日志。

## 新增特性

- 新增结构化终端日志，记录操作时间、客户端 IP、操作类型、任务 ID 及查询结果统计。
- 查询完成日志包含扫描虚拟机数量、命中数量和总耗时，便于快速确认执行结果。
- 新增查询创建、取消请求、参数校验拒绝、查询失败和查询取消等操作日志。

## 优化与修复

- 页面自动轮询查询状态不再输出 `query.status.read` 日志，避免查询过程刷屏。
- 客户端 IP 从与服务直接建立的 TCP 连接地址获取，避免未配置可信反向代理时伪造来源地址。
- 日志不会记录用户名、密码、`X-Auth-Token`、完整请求体或 VRM 返回错误正文。

## 升级说明

- 本版本无需数据迁移或配置调整。
- 升级二进制或镜像后，请重启服务使新的日志行为生效。
- 部署在反向代理后时，日志中的客户端 IP 通常为代理服务器 IP。

详细使用方式见项目根目录的 [README.md](../README.md)。
