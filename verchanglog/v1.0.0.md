# fc-history-query v1.0.0 版本更新日志

**发布日期**：2026-07-18
**版本类型**：首个正式版本

---

## 版本概述

v1.0.0 是 FusionCompute 历史低负载查询的首个正式版本。项目提供一个无需数据库和额外中间件的本地 Web 工具：按需登录 FusionCompute VRM，读取运行中虚拟机的 CPU、内存和磁盘历史指标，计算低负载结论，并支持查询结果二次筛选和格式化 Excel 导出。

## 新增特性

- 新增 FusionCompute VRM 登录、站点发现、运行中虚拟机查询与历史监控指标批量查询链路。
- 新增 CPU、内存、磁盘三类资源指标的最大值、平均值与低负载比例计算。
- 新增“所有采样点均低于阈值”和“低于阈值的采样点比例”两种判定规则。
- 新增虚拟机描述按 `-` 从右拆分用途、管理人、使用期限的展示能力。
- 新增结果表同结论内排序、排序方向选择与 IP 自然排序。
- 新增查询结果全文搜索，以及结论、管理人、集群、主机和资源指标二次筛选。
- 新增格式化 Excel 导出；导出内容与当前筛选后的可见结果保持一致。
- 新增中英文界面切换、浅色/深色主题和查询进度/取消控制。
- 新增 Windows、macOS、Linux 多架构二进制发布与 Docker 镜像发布 CI 流程。

## 安全与运行边界

- 默认仅监听 `127.0.0.1:8088`。
- 用户名、密码和 `X-Auth-Token` 仅保留在当前查询任务内存中；任务完成、失败或取消后清空。
- 查询结果只在服务进程内存中保留 15 分钟，不写入数据库。
- Docker 镜像默认监听容器内 `0.0.0.0:8088`，可通过 `-p` 映射给宿主机访问。

## 部署方式

- GitHub Release：下载对应 Windows、macOS 或 Linux 架构的二进制文件直接运行。
- 源码：使用 Go 1.18+ 执行 `go run ./cmd/server`。
- Docker：使用 `docker run` 运行发布镜像，或从仓库根目录使用 `docker build -f Dockerfile .` 构建本地镜像。
